# Product

## Register

product

## Users

A solo user (the project owner, possibly close collaborators) digitizing handwritten and printed notes for archival or reuse. Sessions are deliberate, short bursts: scan, condition, OCR, copy/export. Used on a desktop browser, often with a tablet or phone alongside as the source.

## Product Purpose

J-Nita v5.0 is an OCR studio that turns photographed pages into clean, structured Markdown. The pipeline is explicit and visible: upload, condition, OCR. Each stage shows its work so the user trusts the result before exporting it.

## Brand Personality

Modern utility, slightly playful — Linear/Raycast school. Confident, instrument-grade, but not cold or corporate. Each pipeline stage has a distinct personality: intake, working, finished. Three words: precise, hands-on, kinetic.

## Anti-references

- **Generic SaaS purple-blue gradients** (`#667eea → #764ba2`). Reads like every AI app from 2023. Banned outright.
- **Stock corporate blue** — predictable navy plus teal of B2B SaaS. Don't lean on these even when "playing it safe".

## Design Principles

- **Color = stage.** Each pipeline step (Input / Process / Result) owns a distinct hue. The user navigates by color memory, not just by labels.
- **Show the seams.** Don't smooth over the multi-step nature. Surfaces, badges, and status indicators wear their stage color so the user always knows which step they are in.
- **Quiet base, loud where it matters.** The page is paper-cream and ink. Color appears only on stage surfaces, status, and primary actions. Everything else stays out of the way.

## Accessibility & Inclusion

WCAG AA contrast for all text. Stage colors are paired with icons and labels — color is never the only signal for state. Stage-tinted surfaces stay below 8% chroma so they read as background, not as alerts.
